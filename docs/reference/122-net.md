# 122 — `libthyla_rs::net`: the native TCP/UDP networking module

**Status**: as-built at net-7a-2 (the native TCP client/server + UDP client API
over `/net`, plus the `sntp` consumer). **Source**:
`usr/lib/libthyla-rs/src/net.rs`; the proving binaries `usr/net-echo` (TCP) and
`usr/sntp` (UDP/SNTP). **Kernel surface**: none (a pure `/net` client).

## Purpose

The native libthyla-rs analog of the pouch AF_INET boundary-line
(`0016-pouch-net-sockets.patch`). Thylacine has **no socket syscalls** by
commitment (ARCH §11.5; NOVEL #1): a native program reaches TCP/IP exactly as a
Plan 9 program does — by walking netd's `/net` connection-server tree
(NET-DESIGN §3.2/§4) with `SYS_open`/`read`/`write`/`close`. This module wraps
that protocol in a `std::net`-shaped API (`TcpStream`, `TcpListener`,
`SocketAddrV4`, `Ipv4Addr`).

It is a **client** of `/net`: a buggy client corrupts only its own state — the
kernel and netd validate every operation. It composes I-1/I-5/I-23/I-28 (a Proc
reaches only the `/net` its territory grants — the namespace firewall, §8) and
adds **no new §28 invariant**.

## Public API

```rust
// Addresses. Surface uses the Rust `a.b.c.d:port` convention; the `!port` Plan 9
// wire form appears only at the netd boundary.
pub struct Ipv4Addr { /* [u8;4] */ }
impl Ipv4Addr {
    pub const LOCALHOST: Ipv4Addr;       // 127.0.0.1
    pub const UNSPECIFIED: Ipv4Addr;     // 0.0.0.0
    pub const fn new(a,b,c,d: u8) -> Ipv4Addr;
    pub const fn octets(&self) -> [u8;4];
    pub const fn is_unspecified(&self) -> bool;
    pub fn parse(s: &str) -> Result<Ipv4Addr>;   // "a.b.c.d", reject malformed
    // impl Display -> "a.b.c.d"
}
pub struct SocketAddrV4 { ip: Ipv4Addr, port: u16 }
impl SocketAddrV4 {
    pub const fn new(ip, port) -> SocketAddrV4;
    pub const fn ip(&self) -> Ipv4Addr;
    pub const fn port(&self) -> u16;
    pub fn parse(s: &str) -> Result<SocketAddrV4>;   // "a.b.c.d:port"
    // impl Display -> "a.b.c.d:port"
}

// Connections.
pub struct TcpStream { /* ctl + data File; both close on drop */ }
impl TcpStream {
    pub fn connect(addr: SocketAddrV4) -> Result<TcpStream>; // blocks to ESTABLISHED
    pub fn read(&mut self, buf) -> Result<usize>;   // blocks (net-6a-1); 0 = EOF
    pub fn write(&mut self, buf) -> Result<usize>;
    pub fn write_all(&mut self, buf) -> Result<()>;
    pub fn shutdown(&mut self) -> Result<()>;       // half-close send (FIN); net-6a-2
    pub fn peer_addr(&self) -> Result<SocketAddrV4>;
    pub fn local_addr(&self) -> Result<SocketAddrV4>;
    pub fn as_raw_fd(&self) -> i32;                 // the data fd (poll/Loom)
    pub fn ready_fd(&self) -> Result<File>;         // the QTPOLL `ready` sibling (net-6b): POLLIN/POLLOUT
    pub fn conn_n(&self) -> u32;
    // impl io::Read + io::Write
}
pub struct TcpListener { /* ctl + N; ctl closes on drop */ }
impl TcpListener {
    pub fn bind(addr: SocketAddrV4) -> Result<TcpListener>;        // announce
    pub fn accept(&self) -> Result<(TcpStream, SocketAddrV4)>;     // blocks
    pub fn local_addr(&self) -> Result<SocketAddrV4>;
    pub fn as_raw_fd(&self) -> i32;                 // the ctl fd
    pub fn conn_n(&self) -> u32;
}

// A connected UDP datagram socket (net-7a; the SNTP client's transport).
pub struct UdpSocket { /* ctl + data File; both close on drop */ }
impl UdpSocket {
    pub fn connect(peer: SocketAddrV4) -> Result<UdpSocket>; // binds a local port
    pub fn send(&mut self, buf) -> Result<usize>;   // one datagram -> the remote
    pub fn recv(&mut self, buf) -> Result<usize>;   // one datagram; BLOCKS (net-6a-1)
    pub fn ready_fd(&self) -> Result<File>;         // the QTPOLL `ready` sibling (net-6b)
    pub fn peer_addr(&self) -> Result<SocketAddrV4>;
    pub fn local_addr(&self) -> Result<SocketAddrV4>;
    pub fn as_raw_fd(&self) -> i32;                 // the data fd
    pub fn conn_n(&self) -> u32;
}

pub struct IcmpSocket { /* ctl + data File; both close on drop */ }
impl IcmpSocket {
    pub fn connect(target: Ipv4Addr) -> Result<IcmpSocket>; // records the ping target (portless)
    pub fn send(&mut self, payload) -> Result<usize>; // one EchoRequest carrying payload
    pub fn recv(&mut self, buf) -> Result<usize>;     // the matching EchoReply payload; BLOCKS
    pub fn ready_fd(&self) -> Result<File>;           // the QTPOLL `ready` sibling (net-6b)
    pub fn as_raw_fd(&self) -> i32;                   // the data fd
    pub fn conn_n(&self) -> u32;
}

// The minimal single-threaded echo server lifecycle (NET-DESIGN §4).
pub fn echo_serve(addr: SocketAddrV4, max_conns: u32) -> Result<u32>;
// A small client slurp helper.
pub fn read_to_vec(stream: &mut TcpStream, cap: usize) -> Result<Vec<u8>>;
```

## Implementation — the `/net` protocol (ground truth: netd's `server.rs`)

| API call | `/net` operations |
|---|---|
| `TcpStream::connect(a:p)` | open `/net/tcp/clone` (ORDWR) → the fid is `ctl`; read it → N; write `connect a.b.c.d!p` to `ctl`; open `/net/tcp/N/data` (ORDWR, blocks to ESTABLISHED) |
| `TcpStream::read`/`write` | read/write the `data` fd (read blocks until bytes or EOF — net-6a-1) |
| `TcpStream::shutdown` | write `hangup` to `ctl` (smoltcp `tcp::close` = a send-side FIN — net-6a-2) |
| `TcpStream::peer_addr`/`local_addr` | read + parse `/net/tcp/N/remote` \| `/local` (`a.b.c.d!p`) |
| `TcpStream::ready_fd` | open `/net/tcp/N/ready` (OREAD) — the QTPOLL readiness sibling (net-6b): `poll(POLLIN)` for a bounded read, or `poll(POLLOUT)` for **write** room — a bulk sender waits POLLOUT before retrying a write that returned 0 (netd's data write is non-blocking; a full 4 KiB send window returns a 0-count `Rwrite`). NP-1's `netperf` M2 is the first consumer |
| `TcpListener::bind(*:p)` | open `clone` → `ctl`/N; write `announce *!p` (or `announce a.b.c.d!p`) |
| `TcpListener::accept` | open `/net/tcp/N/listen` (blocks; the opened fid is the new connection M's `ctl`); read M; open `/net/tcp/M/data`; read M's `remote` for the peer addr |
| `UdpSocket::connect(a:p)` | open `/net/udp/clone` (ORDWR) → `ctl`; read → N; write `connect a.b.c.d!p` to `ctl` (binds a local ephemeral port); open `/net/udp/N/data` (ORDWR) |
| `UdpSocket::send`/`recv` | write/read the `data` fd — each write is one datagram to the connected remote; each read dequeues one datagram (read blocks until one arrives — net-6a-1) |
| `UdpSocket::ready_fd` | open `/net/udp/N/ready` (OREAD) — the QTPOLL readiness sibling, for a **bounded** receive: `poll(POLLIN, timeout)` then `recv` only when it fired (net-6b) |
| `IcmpSocket::connect(a)` | open `/net/icmp/clone` (ORDWR) → `ctl`; read → N; write `connect a.b.c.d` to `ctl` (ICMP is portless — a bare dotted-quad; a 127.x target migrates the socket onto netd's resident loopback stack, which auto-answers in-guest); open `/net/icmp/N/data` (ORDWR) |
| `IcmpSocket::send`/`recv` | write the `data` fd → netd wraps the payload in an EchoRequest (the socket's bound ident + a rotating sequence); read the `data` fd → the matching EchoReply's payload (smoltcp filters replies to the bound ident; read blocks until one arrives — net-6a-1) |
| `IcmpSocket::ready_fd` | open `/net/icmp/N/ready` (OREAD) — the QTPOLL readiness sibling, for a **bounded** receive (the only sane mode for `ping`): `poll(POLLIN, timeout)` then `recv` only when it fired (net-6b) |

The `/net/tcp` and `/net/udp` trees share one set of path helpers parameterized
by the proto string (`clone_conn`/`open_conn_file`/`read_endpoint` take
`"tcp"`/`"udp"`); the TCP behavior is byte-for-byte unchanged.

The clone idiom (Plan 9): opening `clone` mints a fresh connection and rebinds
the opened fid onto that connection's `ctl`; the first read of `ctl` yields the
decimal connection number N. The same fid then takes the `ctl` verbs
(`connect`/`announce`/`hangup`). `accept`'s `listen`-open returns the *new*
connection's `ctl` the same way — the listener stays armed.

### Bounds + error handling

- `Ipv4Addr::parse` / `SocketAddrV4::parse` / `parse_port`: each field length- and
  range-checked; a non-digit, an out-of-range octet/port, or the wrong field
  count returns `InvalidArgument` — never a silent coercion.
- `read_decimal` (the connection-number read): a fixed 16-byte buffer, leading
  ASCII digits only, `checked_mul`/`checked_add` overflow guard; an empty/non-
  numeric reply is `Io`.
- `read_endpoint`: a fixed 48-byte buffer, `trim`, `rsplit_once('!')`, then the
  same bounded address/port parse; any malformation is `Io`.
- **fd lifetime**: `TcpStream`/`UdpSocket` own `ctl` + `data`; `TcpListener`
  owns `ctl`; `accept`'s new `ctl` + `data` move into the returned `TcpStream`.
  `fs::File` closes its fd on `Drop`, so a dropped stream/listener/socket — or any
  error-path early return — leaks no fd and (via the last-clunk-frees-N rule,
  net-2c-1) no netd connection slot. `UdpSocket::ready_fd` returns a fresh
  borrow-free `File`; a caller's `PollSet` registers its raw fd, and the `File`
  must outlive the poll (it does — it is a local that lives to end-of-scope).

## The `sntp` SNTP client (net-7a-2)

`usr/sntp` is the first `UdpSocket` consumer: a native, `ntpdate`-shaped SNTP
(RFC 4330) client that queries an NTP server, computes the wall-clock offset
against the LS-K **monotonic** clock, and steps `CLOCK_REALTIME` via the
net-7a-1 `SYS_CLOCK_SETTIME` (`time::set_realtime`, CAP_HOSTOWNER-gated).

- **`sntp` (no args) / `sntp --selftest`** — the DETERMINISTIC boot gate (no
  I/O): the NTPv4 build/parse/validate battery (mode-4 + stratum∈[1,15] +
  originate-echo anti-spoof + non-zero transmit), the 1900→Unix conversion
  (`ntp_secs − 2_208_988_800`, saturating below the delta — the era-2036 seam),
  the `transmit + rtt/2` offset arithmetic, and the **clock-step gate proof**:
  spawned unelevated, `set_realtime` is denied with `EACCES` — a non-denial is a
  privilege regression, so it fails the boot. Run on every boot by joey.
- **`sntp <a.b.c.d>`** — the LIVE query (best-effort, NOT a boot gate): dial the
  server over `/net/udp`, send a request with a per-request nonce in the
  Transmit field, **bounded-poll** the `ready` sibling for POLLIN (3 s), `recv`,
  validate (the response's Originate must echo the nonce — off-path spoof
  defense), compute the offset, and step the clock (if elevated) or log the
  EACCES. Host-dependent under slirp (no in-guest NTP peer) — the deterministic
  in-guest round-trip over a loopback UDP responder is **owed to net-8**.

The selftest is an **in-guest runtime battery**, not a `cargo test` (a no_std +
aarch64 bin crate — the net-4d `proto_selftest` pattern).

**Trust model (v1.x seam)**: SNTP without authentication trusts the answering
server; the originate-nonce echo defends against *off-path* spoofing (a blind
attacker can't guess it), but an *on-path* attacker is the documented boundary.
NTS / a symmetric-key MAC is the v1.x hardening.

## Tests / proofs

- `net-echo` (the `/net-echo` boot probe, deterministic + peer-independent):
  the parse round-trips + malformed-rejects; `TcpListener::bind`/`local_addr`
  over the live `/net`; and the **Loom async witness** — a Loom `READ` on a
  `/net` fid completes via the kernel dev9p async client, proving network I/O
  rides Loom with **zero new Loom core** (NET-DESIGN §12.1). It deliberately
  does NOT call `accept` (no in-guest peer → it would block).
- `netd`'s `echo_e2e` (server-logic proof): over an isolated 127.0.0.1 loopback,
  announce one listener, accept **two** concurrent clients (the real
  `poll_accepts`/`accept_swap`), echo each payload, verify each client reads
  back its own bytes. This is the server side of what `TcpListener` drives.
- `ping 127.0.0.1` (joey's `net-util PROBE`, deterministic + slirp-independent):
  the full `IcmpSocket` echo path over the live `/net/icmp` mount — clone →
  `connect 127.0.0.1` (migrates onto netd's resident loopback stack, net-8a) →
  `data` write (EchoRequest) → `ready` poll → `data` read (EchoReply). The
  loopback iface auto-answers the echo in-guest, so the round-trip is
  deterministic; ping exits 0 only on a reply, so the probe gates on it.

## Known caveats / seams (net-8)

- **No in-guest live round-trip yet.** netd's live stack is NIC-only (no
  loopback route); net-3d proved the NIC cannot hairpin self-traffic. So a
  native client cannot reach a native server in-guest — the deterministic
  ≥2-concurrent echo proof is the netd-method `echo_e2e`, and the live
  cross-Proc native-API round-trip is **net-8**'s (it owns the peer mechanism:
  a netd loopback interface as its own audited chunk, or bridged QEMU).
- **Single-threaded `echo_serve`.** Accepts and echoes one connection at a time
  (correct for a deterministic send-then-recv client); true interleaved I/O
  across many simultaneously-active streams wants a thread per connection or the
  Loom-multishot async path — net-8.
- **Loom-multishot accept loop** (the literal §12.1 multishot-*read*-on-`listen`)
  wants netd to defer the `listen` *read* (net-3a defers the *open*); the async
  accept composes today via a Loom `LOPEN`-on-`listen`. net-8.
- **IPv4 TCP + connected UDP + ICMP echo.** `TcpStream`/`TcpListener` (net-6a-3) +
  `UdpSocket` (net-7a-2) + `IcmpSocket` (net-utils, the `ping` client). `UdpSocket`
  is *connected* (one remote per socket — the SNTP shape); an unconnected
  `send_to`/`recv_from` over a single `/net/udp` socket is a v1.x add.
- **`IcmpSocket::recv` assumes the first readiness edge is the EchoReply.** A
  reachable host (or the loopback) answers with an EchoReply, so `ready_fd`'s
  POLLIN → `recv` returns the payload. But an ICMP *error* response (e.g.
  Destination Unreachable) that quotes our ident also makes the socket readable;
  `recv` consumes it, finds it is not an EchoReply, and blocks for the *next*
  packet — so pinging a host that sends only an ICMP error waits (interruptibly —
  the read is #811 death-interruptible, so Ctrl-C frees it) for a reply that does
  not come. The silent-host case is bounded by the 1 s poll timeout; only the
  error-but-no-echo case waits. A per-`recv` deadline (composing the TLS-deadline
  seam, net-8d F2) is the v1.x fix — task #256.
- `SocketAddrV4` only (no IPv6 / hostnames; resolution is `/net/cs`+`/net/dns` —
  `sntp` takes a dotted-quad at v1.0).
- **SNTP step accuracy** is `transmit + rtt/2` (the SNTP simple algorithm), not a
  disciplined PLL; it steps, it does not slew. A drift-correcting daemon is a
  v1.x add.
