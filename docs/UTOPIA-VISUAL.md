# UTOPIA-VISUAL — Pale Fire

Thylacine's user-visible visual identity scripture. Binding for every Utopia program that emits coloured output. Consumed by `libutopia::palette` and the `ut` shell's prompt-emit path. Halcyon (Phase 8) renders the same identity natively on Thylacine's own framebuffer; until then the host terminal renders it.

**STATUS**: COMMITTED — scripture commit, U-1 chunk.

---

## 1. The palette — *Pale Fire*

Named after the Nabokov novel. Cold near-black background; moonlight foreground; a single ember glyph as the only warm note.

| Role | Hex | Description |
|---|---|---|
| Background | `#0e1018` | Cold near-black, faint blue cast. |
| Foreground | `#d8e4f4` | Moonlight — all text, all output. |
| Prompt path | `#8898b4` | Receded steel; path segment only. |
| Glyph `⊢` | `#e07840` | Ember orange; the `⊢` turnstile only. |

These are the only four colours in v1. Output, errors, completions, and interactive prompts all use foreground (`#d8e4f4`) without exception in the disciplined programs.

The two-tier text hierarchy — path vs. everything else — is the only distinction Utopia's own tools draw. Errors are not colour-coded: they are context-coded (preceded by the program name, or a structured prefix per ERRORS.md). Disciplined Utopia programs (the shell, the coreutils, libutopia-consumers) do not introduce additional colours.

User programs are free to colour however they like; the discipline applies to the Utopia surface, not to third-party code.

## 2. The `⊢` glyph — U+22A2 RIGHT TACK

The prompt glyph. Unicode codepoint `U+22A2` RIGHT TACK (the turnstile). The byte sequence in UTF-8 is `E2 8A A2`.

Rationale for the choice:

- **Formal meaning.** In sequent calculus and proof theory, `⊢` reads as "proves" or "yields." The shell yields commands to the kernel; every invocation is a derivation. The glyph names the act.
- **Visual reading.** A horizontal T — for Thylacine.
- **Uniqueness in terminal-land.** No widely-used shell uses this character as its prompt glyph. It is unoccupied, so it carries Thylacine's identity uniquely.
- **Renders at all sizes.** A simple two-stroke shape; clean in any monospace typeface, distinguishable at the smallest terminal sizes.

The glyph appears in foreground colour `#e07840` (ember orange) — the only warm note in the otherwise-cold palette.

## 3. Prompt format

```
~/src/thylacine ⊢ command argument
^─────────────^ ^ ^──────────────^
  #8898b4       │     #d8e4f4
              #e07840
```

Three segments:

| Segment | Colour | Content |
|---|---|---|
| Path | `#8898b4` | The current working directory, abbreviated with `~` for `$home`. |
| Glyph | `#e07840` | Single character `⊢` flanked by single spaces. |
| Command | `#d8e4f4` | The user's editable command line. |

The path recedes. The glyph is the warm hinge. The command is the content.

### 3.1 Path abbreviation

- Exactly `$home` → `~`.
- A path under `$home` → `~/<relative>`.
- A path under a bound namespace name (per `bind`/`mount` builtins) → the bound name as the path prefix; the shell's tracking of the binding is the source of truth.
- An absolute path elsewhere → the full path, with `…` truncating the middle if the path exceeds a terminal-width-derived budget.

### 3.2 Multi-line continuation

When the command line wraps or continues (unbalanced brackets, trailing `\`), subsequent input lines use a continuation glyph in place of the path:

```
~/src/thylacine ⊢ {
                ⋮ for (f in *.md) {
                ⋮     wc -l $f
                ⋮ }
                ⋮ }
```

Continuation glyph: `⋮` (U+22EE VERTICAL ELLIPSIS) at `#8898b4` (the receded path colour). Indented so the next character aligns under the original command start.

### 3.3 Prompt as a function

The prompt is generated by the `prompt` shell function, declared in `~/.config/utopia/utopia.rc`. The shell calls `prompt` before each readline cycle and emits its captured output. The default `prompt` in the shipped `utopia.rc` is:

```
fn prompt {
    palette path (pwd)
    palette glyph ' ⊢ '
}
```

Users override `prompt` to add segments (git branch, namespace name, host indicator, etc.); their additions are not bound by the Utopia palette discipline.

## 4. The `palette` builtin and `libutopia::palette`

The `palette` shell builtin and the matching `libutopia::palette` Rust module are the canonical entry points for emitting palette-coloured text. They handle:

- ANSI 24-bit colour escape emission (`ESC[38;2;R;G;Bm` foreground, `ESC[0m` reset).
- The `length-doesn't-count` markers that the line editor needs for cursor accounting (so the prompt's actual visible width equals the visible character count, not the byte count).
- The colour names → RGB resolution table.
- The trailing reset after every coloured emission, so a programmer cannot accidentally bleed colour into subsequent output.

### 4.1 Colour names

Programs refer to colours by role, not hex. The role table is the single source of truth; the hex values can change in v1.x (they will not in v1) without sweeping every emission site.

| Role name | Constant in `libutopia::palette` | Hex |
|---|---|---|
| `bg` | `palette::BG` | `#0e1018` |
| `fg` | `palette::FG` | `#d8e4f4` |
| `path` | `palette::PATH` | `#8898b4` |
| `glyph` | `palette::GLYPH` | `#e07840` |

### 4.2 Shell-builtin shape

```
palette ROLE TEXT...
```

Emits `TEXT...` (concatenated with spaces) in the named role's colour, followed by a reset. The line editor's prompt-length accounting treats the ANSI escapes as zero-width.

### 4.3 Rust API shape

```rust
// in libutopia::palette
pub fn emit_fg(role: Role, text: &str);   // foreground in role's colour + reset
pub fn emit_bg(role: Role, text: &str);   // background in role's colour + reset
pub fn fg_seq(role: Role) -> &'static str; // raw ANSI escape; caller emits its own reset
pub fn reset() -> &'static str;            // ESC[0m
```

`Role` is the enum named after the role table in §4.1.

### 4.4 Capability detection

V1 does not attempt terminal-capability detection. The escape sequences are emitted unconditionally; terminals that don't support 24-bit colour will degrade in their own way (typically by ignoring the escape and printing the text plainly; some terminals print the raw escape as garbage). Users running Thylacine via UART or SSH are expected to use a 24-bit-colour-capable terminal (Ghostty, iTerm2, Kitty, foot, Alacritty, WezTerm — all of which support 24-bit colour by default).

Capability detection (downgrade to 8-colour, fallback to no colour) is a v1.x consideration.

## 5. Typeface

The recommended typeface for the Utopia experience is **PragmataPro Mono**. Its narrow-condensed metrics, comprehensive Unicode coverage including U+22A2, and ligature support match Pale Fire's intent.

Distribution is the user's responsibility at v1.0. PragmataPro is commercial software (FSD Type Foundry; see https://fsd.it/shop/fonts/pragmatapro/). Thylacine does not bundle it. Users who hold a PragmataPro license configure their host terminal to use it.

A free, open-source fallback typeface — likely **Iosevka** (OFL) — will be selected and configured as Halcyon's bundled default in Phase 8 design. Iosevka shares PragmataPro's narrow-condensed metrics and has excellent Unicode coverage including the relevant glyphs.

Until Halcyon ships, the visual identity expresses through ANSI bytes sent to the host terminal; the host's font choice is up to the user. Thylacine ships terminal configuration files (Ghostty / Kitty / Alacritty / iTerm2) at `share/terminal-configs/` that pin Pale Fire colours and recommend the typeface choice.

## 6. Sixel and inline graphics

Sixel (DEC's inline-image escape sequence) and the Kitty graphics protocol are how programs embed images inline in terminal output. Thylacine programs MAY emit Sixel and/or Kitty-graphics escapes; the host terminal renders them (Ghostty, Kitty, foot, WezTerm, iTerm2 all support at least one of the two).

Sixel emission is **deferred from U-1 scripture** and tracked as a Utopia v1.x extension: when a program (e.g., `9 fortune`, a future image-preview command) wants inline graphics, it grows the capability. The infrastructure is just bytes-out.

Halcyon (Phase 8) implements the **decoder** side of Sixel and Kitty graphics, so that the same programs continue to work natively when Thylacine owns the screen.

## 7. Discipline summary

Bullet list for fast review:

- Four colours only at v1: `bg`, `fg`, `path`, `glyph`.
- The path segment is the only `path`-coloured text; everything else is `fg` except the `⊢` glyph.
- The `⊢` glyph is the only `glyph`-coloured text.
- Errors are not colour-coded.
- Continuations use `⋮` in `path` colour.
- Programs reach the palette via `libutopia::palette` (Rust) or the `palette` shell builtin.
- Discipline applies to Utopia's own programs; third-party tools colour as they wish.
- No terminal capability detection at v1; v1.x can add it.
- No Sixel at U-1; v1.x can add it.
- No bundled font at v1; the user configures their host terminal.
