## What landed

### P3-Dc: demand paging

**`mmu_install_user_pte(pgtable_root, asid, vaddr, pa, prot)`** in `arch/arm64/mmu.c` (~120 LOC):
- Walks L0 → L1 → L2 → L3 of the per-Proc TTBR0 tree.
- Allocates missing sub-tables via `alloc_pages(0, KP_ZERO)` and installs table descriptors.
- Installs an L3 leaf page descriptor at the appropriate index with bits derived from VMA prot:
  - AP[2:1]: AP_RW_ANY for VMA_PROT_WRITE else AP_RO_ANY (no AP_RW_EL1 — user PTEs are always EL0-accessible).
  - UXN: clear for VMA_PROT_EXEC else set.
  - PXN: unconditionally set (kernel never executes user pages).
  - AF + nG + SH_INNER + ATTR_IDX(NORMAL_WB) standard.
- Argument validation: zero pgtable_root, unaligned vaddr/pa, high-VA vaddr (TTBR1 territory), out-of-IPS pa, W+X prot all return -1.
- Idempotent on identical re-install; mismatching re-install (different PA at same vaddr) returns -1.
- No TLB flush issued; invalid → valid transitions don't require it (ARM ARM B2.7.1). DSB ISHST + ISB at the end ensures the new PTE is observable to the MMU walker before ERET.
- W^X invariant I-12 holds by construction at the PTE level: AP_RW_ANY only ever pairs with UXN set; UXN clear only ever pairs with AP_RO_ANY.

**`userland_demand_page(Proc*, fault_info*)`** in `arch/arm64/fault.c` (~50 LOC):
- vma_lookup(p, fi->vaddr) — sorted-list walk to find the covering VMA. Returns NULL → FAULT_UNHANDLED_USER.
- Permission check: write fault requires VMA_PROT_WRITE; instruction fault requires VMA_PROT_EXEC; read fault requires VMA_PROT_READ.
- Resolve VMO offset: `vmo_byte_off = vma->vmo_offset + (page_va - vma->vaddr_start)`. Reject if ≥ `vmo->size`.
- Resolve to backing PA: `vmo_base_pa + (vmo_byte_off & ~PAGE_MASK)` (v1.0 anonymous VMO: pages contiguous starting at `vmo->pages`).
- mmu_install_user_pte → returns FAULT_HANDLED on success.

**`arch_fault_handle`'s user-mode case** rewired:
- Was: `if (fi->from_user) return FAULT_UNHANDLED_USER;`
- Now: validates current_thread + Proc magic, then calls `userland_demand_page(t->proc, fi)`.

### Tests added (7 new, in `kernel/test/test_demand_page.c`)

- `pgtable.install_user_pte_smoke`: install RW + RX leaf PTEs; walk the tree manually; verify all PTE bits (AF, nG, PXN, UXN, AP encoding); verify backing PA matches.
- `pgtable.install_user_pte_constraints`: zero pgtable_root, unaligned vaddr/pa, high-VA vaddr, W+X prot — all return -1.
- `pgtable.install_user_pte_idempotent`: identical re-install returns 0 + does NOT allocate (verified via phys_free_pages snapshot); mismatching re-install returns -1.
- `demand_page.smoke`: synthetic fault_info on a mapped VMA → FAULT_HANDLED + L3 PTE installed pointing at the VMO's first page.
- `demand_page.no_vma`: fault on unmapped VA → FAULT_UNHANDLED_USER.
- `demand_page.permission_denied`: write fault on RO VMA / instruction fault on non-EXEC VMA → FAULT_UNHANDLED_USER. Read fault on R-only VMA succeeds.
- `demand_page.lifecycle_round_trip`: 4-page VMA + demand-page each + proc_free + vmo_unref → phys_free_pages returns to baseline. Validates the full lifecycle: sub-tables freed by P3-Db walker; backing pages freed by VMO refcount lifecycle.

### Spec posture

No new TLA+ spec at P3-Dc. Rationale (documented in 25-fault-dispatcher.md and inline in fault.c):

- **No new concurrency**: under the v1.0 single-thread-Proc invariant from P2-D, only the running thread of Proc P faults P's pgtable. No two CPUs concurrently demand-page on the same Proc.
- **No new refcount semantics**: the demand-page path doesn't take or release VMO refs. The VMA already holds `mapping_count`. specs/vmo.tla::NoUseAfterFree continues to hold.
- **W^X invariant by construction**: VMA prot already excludes W+X (vma_alloc rejects); PTE bit derivation in mmu_install_user_pte preserves this; the defensive W+X reject in install is a third layer.

Phase 5+ multi-thread Procs introduce concurrent fault handlers on different threads of the same Proc → race on sub-table allocation. That's the spec-extension point.

## Trip-hazards added at P3-Dc (cumulative #136-141)

136. **`mmu_install_user_pte` is NOT serialized at v1.0** — single-thread Procs invariant. Phase 5+ multi-thread MUST add a per-Proc pgtable lock around the walk-allocate-install sequence.

137. **PTE encoding for user pages**: AP_RW_ANY for VMA_PROT_WRITE else AP_RO_ANY; UXN clear for VMA_PROT_EXEC else set; PXN unconditionally; nG; AF; SH_INNER; ATTR_IDX(NORMAL_WB).

138. **Demand-paged backing PA = `page_to_pa(vmo->pages) + (vmo_byte_off & ~PAGE_MASK)`** at v1.0 anonymous VMOs. Phase 5+ VMO_TYPE_FILE will need per-page lookup.

139. **`userland_demand_page` is exposed in fault.h for tests** — synthetic fault_info + manually-constructed Proc bypasses the need to trigger a real EL0 fault (impossible at v1.0 pre-exec).

140. **mmu_install_user_pte is idempotent on identical re-install** but mismatch returns -1. Phase 5+ COW will need a different protocol.

141. **Per-Proc pgtable walker reads via pa_to_kva** (both install + destroy). Doesn't touch TTBR0_EL1; the Proc need not be currently scheduled.

## Closing notes

P3-Dc closes the demand-paging side of the user-mode address-space pipeline. The full chain `vmo_create_anon → vmo_map → vma_insert → fault → vma_lookup → mmu_install_user_pte → ERET-resume` is now wired and exercised by 7 unit tests + 1 lifecycle round-trip test. v1.0 pre-exec means demand paging is exercised by tests only, but the pipeline is complete and ready for P3-E.

P3-E (exec syscall) is the next milestone. It needs:
- `exception_sync_lower_el` to wire the EL0 sync exception vector to the same `arch_fault_handle` dispatcher.
- A user-stack VMO + initial mapping.
- The exec dispatcher itself: parse ELF (P2-Ga), per-segment vmo_create + vmo_map, set up user stack, ERET to entry point with PSTATE = EL0t.
- Probably a TLA+ spec for the exec atomicity (parse failure + map failure rollback semantics).
- A demonstrator: a static ELF that does a "hello" via syscall (and the syscall infrastructure to print).
