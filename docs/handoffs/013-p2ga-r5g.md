### P2-Ga — ELF64 ARM64 loader (parse + W^X validation)

`7e2598b` substantive + `7c333d8` hash fixup.

- New `kernel/include/thylacine/elf.h` (~210 LOC) + `kernel/elf.c` (~150 LOC) + `kernel/test/test_elf.c` (~250 LOC) + `docs/reference/21-elf.md`.
- **Pure-parsing + validation library**. Inspects an ELF blob in memory, rejects malformed or W^X-violating inputs, produces a structured segment list. Mapping deferred to Phase 3 (when address spaces + page-fault handler land); `exec()` syscall surface deferred to Phase 5+.
- **5-stage validation pipeline**: e_ident → e_type/e_machine/e_version → phdr-table-layout (with overflow-safe phoff + phnum*phentsize) → per-segment validation (W^X / bounds / filesz/memsz) → entry-in-segment.
- **Overflow-safe arithmetic** at every additive size check (`u64_add_overflow`, `u32_mul_widen`).
- **W^X enforcement** at the loader layer: `(p->p_flags & (PF_W | PF_X)) == (PF_W | PF_X)` rejected. ARCH §28 I-12. Third enforcement layer alongside PTE bits (`mmu.c`) and `mprotect` (Phase 5+ syscall).
- **Static-only policy at v1.0**: `PT_INTERP` rejected (was at landing); `PT_DYNAMIC` rejected (added at R5-G F63 close).
- **NX stack**: `PT_GNU_STACK` with `PF_X` rejected.
- **Sane bounds**: `phnum ≤ 256`; `phtab + segments` within blob size; `filesz ≤ memsz`; entry within some LOAD segment.
- 6 new tests (post-R5-G additions in **bold**): `parse_minimal_ok` / `parse_multi_segment_ok` (text RX + rodata R + data RW) / `header_rejection` (magic / class / data / version / OSABI / type ET_REL+ET_DYN / machine / phentsize) / `rwx_rejected` (PF_R+PF_W+PF_X + PF_W+PF_X both rejected; RX/RW + OS-specific bits accepted) / `bounds_rejection` (TOO_SMALL/NULL/no-phdrs/too-many-phdrs/phtab-OOB/filesz>memsz/seg-OOB **+ misaligned blob (R5-G F61)** **+ misaligned phoff (R5-G F62)** **+ TOO_MANY_LOADS (R5-G F68)**) / `policy_rejection` (PT_INTERP **+ PT_DYNAMIC (R5-G F63)** + PT_GNU_STACK with PF_X / entry==0 / entry-outside / **+ entry boundary tests (R5-G F69)** + final positive control).

## Trip-hazards added since handoff 012

### P2-Ga (ELF loader)

(See handoff 012's P2-Ga listing — those are the substantive trip-hazards. Below are R5-G-introduced clarifications.)

### R5-G audit close

- **Blob alignment now mandatory at the API surface**. Callers passing slices into other buffers must ensure 8-byte alignment — kernel kmalloc / page-allocator memory satisfies this trivially, but a future caller using a non-aligned source must `memcpy` to an aligned buffer first. New error `ELF_LOAD_BAD_ALIGN`.
- **e_phoff must be 8-byte aligned**. Real linkers always emit `e_phoff = 64`, naturally aligned. An attacker-crafted misalignment is rejected at parse time.
- **PT_DYNAMIC explicitly rejected**. Future Phase 5+ dynamic-linker support flips this case from "reject" to "process the dynamic table." The error code is dedicated (`ELF_LOAD_HAS_DYNAMIC`) so the future-flip is greppable.
- **W^X check is type-blind**. Any future PT_* type added to the kernel inherits W^X enforcement automatically. The check is no longer a per-case responsibility.
- **`*out` zeroed on failure**. Defensive; defense-in-depth against buggy callers.
