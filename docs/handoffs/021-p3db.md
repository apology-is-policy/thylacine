## What landed

### P3-Db: vmo_map signature extension + proc_pgtable_destroy walk

#### API split

The bare refcount-only ops are **renamed**:

| Pre-P3-Db | Post-P3-Db |
|---|---|
| `void vmo_map(struct Vmo *v)` | `void vmo_acquire_mapping(struct Vmo *v)` |
| `void vmo_unmap(struct Vmo *v)` | `void vmo_release_mapping(struct Vmo *v)` |

The names `vmo_map` / `vmo_unmap` are now used for the public high-level entries:

```c
int vmo_map(struct Proc *p, struct Vmo *v,
            u64 vaddr, size_t length, u32 prot);
int vmo_unmap(struct Proc *p, u64 vaddr, size_t length);
```

`vmo_map(Proc*, ...)` validates args (NULL/zero/alignment/W+X/overflow) → `vma_alloc(...)` (which takes `vmo_acquire_mapping` internally) → `vma_insert(p, vma)`. On `vma_insert` overlap (returns -1), `vma_free(vma)` rolls back the acquire. Returns 0/-1.

`vmo_unmap(Proc*, ...)` validates args → `vma_lookup(p, vaddr)` → exact-match check (vaddr_start AND vaddr_end) → `vma_remove + vma_free`. Returns 0/-1. v1.0 requires exact-match; partial unmap is post-v1.0.

PTE installation is NOT performed — demand paging via P3-Dc populates the per-Proc TTBR0 tree on fault.

#### `proc_pgtable_destroy` recursive walk

Pre-P3-Db only freed the L0 page (P3-Bcb baseline; documented as trip-hazard #116). At P3-Db extended to walk L0 → L1 → L2 → L3 freeing every reachable sub-table.

Implementation in `arch/arm64/mmu.c` (~50 LOC):
- `l3_walk_and_free(l3_pa)`: free the L3 page itself; do NOT touch the 512 leaf entries (those reference VMO-backed user pages owned by the VMA layer).
- `l2_walk_and_free(l2_pa)`: walk 512 entries; for table descriptors recurse into L3; for block descriptors (2-MiB user blocks) skip; free the L2 page.
- `l1_walk_and_free(l1_pa)`: same shape as `l2_walk_and_free` but for 1-GiB blocks.
- `proc_pgtable_destroy(root)`: walk 512 L0 entries (only table descriptors valid at v1.0); recurse into L1; free the L0 page.

The walker reads PTE bits via `pa_to_kva(table_pa)` (kernel direct-map alias); it doesn't touch TTBR0. ASID-tagged TLB lifecycle is handled independently by `asid_free`'s broadcast `tlbi aside1is`. Walker-vs-asid_free order is irrelevant for correctness — `proc.c::proc_free`'s order (pgtable_destroy, then asid_free) is correct, and reversing would also be correct.

### Tests added

**`kernel/test/test_vmo_map_proc.c` (new file, 5 tests, ~165 LOC)**:
- `vmo.map_proc_smoke`: vmo_map installs a VMA visible via vma_lookup; mapping_count tracks; vma_drain releases to baseline.
- `vmo.map_proc_constraints`: NULL Proc/VMO, zero length, unaligned vaddr/length, W+X prot — all return -1; mapping_count untouched; no VMA installed.
- `vmo.map_proc_overlap_rejected`: adjacent (touching boundary) accepted; exact + partial overlap rejected; rollback verified via mapping_count delta.
- `vmo.unmap_proc_smoke`: exact-match unmap removes the VMA + decrements mapping_count.
- `vmo.unmap_proc_no_match`: non-matching range / unaligned start / wrong length all return -1.

**`kernel/test/test_proc_pgtable.c` (extended; 1 new test)**:
- `proc.pgtable_destroy_walk_releases_subtables`: manually install a 3-deep sub-table chain (L1 → L2 → L3); proc_free; verify `phys_free_pages()` returns to baseline (all 4 pages: L0+L1+L2+L3 freed). Pre-P3-Db's L0-only free would leave free count 3 short.

## Trip-hazards added at P3-Db (cumulative #131-135)

131. **Renamed: `vmo_map(Vmo*)` → `vmo_acquire_mapping`; `vmo_unmap(Vmo*)` → `vmo_release_mapping`**. Bare refcount-only ops keep semantics; only names changed. test_vmo.c uses the new names.

132. **`vmo_map(Proc*, ...)` rolls back on `vma_insert` overlap rejection**. The rollback is in `vmo_map` itself, not at the caller.

133. **`vmo_unmap(Proc*, ...)` requires exact-match at v1.0**. Partial unmap is post-v1.0.

134. **`proc_pgtable_destroy` walks L0 → L1 → L2 → L3 freeing every reachable sub-table** (closes trip-hazard #116). Leaf descriptors are NOT followed; their pages belong to the VMA layer.

135. **`proc_pgtable_destroy` ordering vs `asid_free`** is irrelevant for correctness — both orders are safe.
